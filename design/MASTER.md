## 가장 중요한 원칙
현재 구조는 절대 바꾸지 않는다. 레퍼런스 이미지는 "레이아웃 복제"가 아니라 "시각 스타일 레퍼런스"로만 사용한다.

즉, 아래 항목은 유지한다.
- 현재 화면 구조
- 현재 정보 계층
- 현재 탭/섹션 구성
- 현재 사용자 플로우
- 현재 컴포넌트 배치 논리

아래 항목만 레퍼런스에 가깝게 바꾼다.
- 색상
- 테두리
- 폰트 인상
- 버튼 질감
- 카드 표면 톤
- 개미 캐릭터의 재질감과 표정 방향
- 게이지, 말풍선, 아이콘의 시각 어휘

## 이번 문서의 목적
이 문서는 현재 AntGravity의 구조를 유지한 채, 레퍼런스 이미지가 가진 프리미엄 브라운 톤, 허니 골드 포인트, 묵직한 타이포그래피, 3D 개미 캐릭터 감성을 입히기 위한 시각 규칙을 정리한다.

## 구조 보존 해석
레퍼런스 이미지에는 데스크톱 대시보드형 구성이 보이지만, 이 프로젝트는 현재 구조를 유지해야 한다. 따라서 이번 작업에서는 레퍼런스의 아래 요소만 추출한다.

1. 아이보리 기반의 따뜻한 배경색
2. 짙은 브라운 외곽선
3. 꿀 색상의 포인트 컬러
4. 두껍고 둥근 인상의 한글 타이포그래피
5. 3D 개미 캐릭터와 말풍선 중심의 정보 전달 방식

추출하지 않는 요소는 아래와 같다.
- 데스크톱 전용 2행 3열 대시보드 구조
- 좌측 고정 사이드바
- 레퍼런스와 동일한 카드 배치

## 비주얼 방향

### 전체 무드
- 키워드: 따뜻함, 프리미엄, 묵직함, 귀여움, 수공예감
- 차가운 금융 앱보다 세계관이 있는 생존형 투자 앱처럼 보여야 한다
- 흰색 중심의 평면 UI보다 크림, 브라운, 골드 조합이 먼저 읽혀야 한다

### 피해야 할 것
- 얇은 회색 보더
- 푸른빛 금융 앱 컬러
- 이모지로만 해결한 캐릭터 표현
- 너무 가벼운 기본 시스템 폰트 느낌
- 차갑고 기계적인 표면 질감

## 색상 시스템

### 코어 팔레트
| 토큰 | 색상 | 용도 |
|------|------|------|
| `Canvas Ivory` | `#F6F1E7` | 전체 배경 |
| `Canvas Warm` | `#EFE4D2` | 보조 배경, 카드 안쪽 온기 |
| `Card Cream` | `#FCF8F1` | 카드 기본 배경 |
| `Stroke Brown` | `#5E3A23` | 핵심 외곽선, 제목 |
| `Stroke Brown Deep` | `#3D2416` | 더 강한 텍스트, 강조 보더 |
| `Text Secondary` | `#6E4C39` | 보조 텍스트 |
| `Honey Gold` | `#F1B640` | CTA, 상태 게이지, 강조 |
| `Honey Deep` | `#C9831F` | 골드 음영 |
| `Leaf Green` | `#6AA84F` | 상승/긍정 |
| `Bear Red Brown` | `#A45B3E` | 하락/경고 |
| `Soft Line` | `#D7C1A1` | 약한 분리선 |

### 사용 원칙
- 기존 레이아웃은 유지하되, 배경과 카드 표면을 더 따뜻한 크림 계열로 바꾼다
- 외곽선은 회색 대신 브라운 계열을 사용한다
- 강조색은 주황보다 허니 골드 쪽으로 옮긴다
- 상승/하락 색은 형광이 아니라 흙빛이 살짝 섞인 톤으로 정리한다

## 보더와 표면

### 외곽선 규칙
- 현재 카드, 버튼, 입력창 구조는 유지
- 다만 외곽선 시각어는 레퍼런스처럼 더 선명한 브라운으로 맞춘다
- 핵심 컨테이너는 2px~3px 브라운 외곽선을 우선 사용한다
- 보조 라인은 1px `Soft Line` 계열로 정리한다

### 반경 규칙
| 요소 | 권장값 |
|------|--------|
| 카드 | 20px~28px |
| 버튼 | 9999px 또는 18px |
| 말풍선 | 20px~24px |
| 입력창 | 16px~20px |
| 아바타 | 9999px |

### 표면 질감
- 카드 배경은 완전한 새하양보다 크림 화이트를 우선한다
- 브라운 카드가 필요한 경우 기존 기능 구조는 유지하되 강조 영역에만 사용한다
- 그림자는 깊지 않게 넣고, 외곽선 존재감이 더 커야 한다

## 타이포그래피

### 서체 방향
레퍼런스 원본 폰트명은 확정할 수 없으므로, 구현 기준은 "둥글고 묵직한 한글 산세리프"로 정의한다.

### 권장 스택
1. `Pretendard Variable`, `Pretendard`
2. `Noto Sans KR`
3. `sans-serif`

### 타입 인상
| 용도 | 크기 방향 | 굵기 방향 | 원칙 |
|------|-----------|-----------|------|
| 메인 타이틀 | 기존보다 약간 큼 | 800~900 | 브라운 고정 |
| 카드 타이틀 | 기존보다 큼 | 700~800 | 묵직하게 |
| 본문 | 현재 구조 유지 | 500~700 | 가독성 우선 |
| 라벨/수치 | 현재 구조 유지 | 600~800 | 상태 전달 명확히 |

### 문자 스타일 원칙
- 자간을 과하게 벌리지 않는다
- 숫자와 한글이 같이 있어도 시각 무게가 맞아야 한다
- 버튼 텍스트는 흰색보다 짙은 브라운을 우선 검토한다

## 캐릭터 시스템

### 캐릭터 사용 원칙
- 현재 캐릭터 역할 체계는 유지한다
- 캐릭터의 표현 방식만 레퍼런스처럼 3D 장난감 질감으로 가깝게 만든다
- 몸통은 브라운, 볼은 살구빛, 악세사리는 골드 톤으로 통일한다

### 캐릭터 정의
| 캐릭터 | 역할 | 비주얼 포인트 |
|--------|------|--------------|
| 병정개미 | 리스크/상태 전달 | 브라운 몸통, 장비 포인트, 단단한 표정 |
| 여왕개미 | AI 분석/가이드 | 왕관, 둥근 얼굴, 따뜻한 미소 |
| 정찰개미 또는 커뮤니티 개미 | 피드/보조 안내 | 원형 아바타, 귀여운 표정, 브라운 쉘 |

### 금지 사항
- 이모지 자체를 캐릭터 완성형으로 쓰지 않는다
- 플랫 2D 벡터 느낌으로 단순화하지 않는다
- 캐릭터마다 전혀 다른 컬러 세계를 주지 않는다

## 컴포넌트 시각 가이드

### 카드
- 현재 카드 종류와 위치는 유지
- 배경은 `Card Cream`
- 보더는 `Stroke Brown`
- 카드 헤더는 더 두껍고 진하게

### 버튼
- 현재 버튼 구조와 동작은 유지
- 색상만 허니 골드 기반으로 조정
- 둥근 형태와 하단 음영으로 꿀 같은 무게감을 준다

### 입력창
- 현재 입력창 구조는 유지
- 배경은 크림 톤
- 보더는 브라운
- 검색/입력 아이콘은 얇지 않게

### 말풍선
- 현재 말풍선 구조는 유지
- 꼬리와 외곽선을 브라운으로 정리
- 내부는 따뜻한 화이트

### 게이지/상태바
- 현재 기능 구조는 유지
- 시각 표현만 브라운 트랙 + 허니 골드 포인트로 통일
- 상승/하락 컬러는 초록/적갈색 보조로만 사용

## 최종 원칙
이번 작업의 목표는 "구조 변경"이 아니라 "시각 레이어 교체"다. 구현과 문서 모두 현재 구조를 고정한 상태에서, 레퍼런스 이미지의 색상, 테두리, 폰트 인상, 캐릭터 재질감을 최대한 가깝게 옮기는 데 집중해야 한다.
