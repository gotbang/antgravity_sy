# AntGravity Supabase Direct-Read PRD

## 작성시각
2026-03-22 16:20:06 KST

## 문서 목적
이 문서는 AntGravity 주식 모드의 현재 제품 요구사항을 direct-read 기준으로 고정한다. 이번 갱신에서는 먹이 탐색 가격 공백을 상태 중심 UX로 처리하고, `안전 활동반경`을 실데이터 기반 가이드로 포함한다.

## 1. 제품 배경
- 현재 주식 모드는 정적 프론트에서 Supabase 공개 뷰를 직접 읽는 구조다.
- FastAPI 읽기 경로는 rollback-only 문맥이다.
- 사용자에게 중요한 건 오늘 탭과 먹이 탐색 탭이 빠르고 안정적으로 보이는지다.

## 2. 해결하려는 문제
- 검색 가능한 종목과 실제 가격을 가진 종목의 기준이 달라 카드에서 빈 가격이 나온다.
- 가격 빈칸은 “데이터가 없는가, 로딩이 실패했는가”를 구분하기 어렵게 만든다.
- `안전 활동반경`이 고정 UI라서 실제 시장 상황과 연결된 느낌이 약하다.

## 3. 핵심 사용자 가치
- 오늘 탭에서 시장 요약과 리스크 점수를 빠르게 확인할 수 있다.
- 먹이 탐색에서 가격이 없어도 빈칸 대신 명확한 상태를 본다.
- 카드에서 활동반경을 실제 적재 데이터 기반으로 이해할 수 있다.
- 브라우저는 단순한 direct-read 구조를 유지하고, 데이터 품질 문제는 운영 이슈로 분리한다.

## 4. 대상 사용자
- 오늘 시장 분위기를 빠르게 확인하려는 사용자
- 종목명을 검색해 대표 카드 한 장으로 판단하려는 사용자
- 정교한 트레이딩 툴보다 요약 중심 탐색을 원하는 사용자

## 5. 현재 범위

### 포함
- `v_home_summary` 기반 오늘 탭
- `v_stock_search` 기반 검색 결과 리스트
- `v_stock_detail_latest` 기반 단일 종목 카드
- `price_status` 기반 가격 상태 표시
- `safe_activity_radius_pct` 기반 활동반경 표시
- 기본 심볼 `AAPL`
- 검색 결과 선택 후 동일 카드 갱신

### 제외
- 트렌딩 UI
- 생존 일지 저장/복원
- 로그인/계정
- 실시간 시세 스트리밍
- 브라우저 단계 라이브 폴백

## 6. 핵심 사용자 흐름

### 흐름 1. 오늘 탭 확인
1. 사용자가 주식 모드에 들어간다.
2. 화면은 `v_home_summary`에서 시장 기분, 공포탐욕 점수, 상승/하락 비중, AI 요약을 읽는다.
3. 조회 실패 시 화면은 가짜 숫자를 유지하지 않고 기본 문구를 유지한다.

### 흐름 2. 기본 카드 확인
1. 사용자가 먹이 탐색 탭을 연다.
2. 기본 카드 한 장이 `AAPL` 기준으로 표시된다.
3. 카드 가격과 활동반경은 `v_stock_detail_latest` 계약에 따라 렌더된다.

### 흐름 3. 검색 후 종목 교체
1. 사용자가 종목명이나 심볼을 입력한다.
2. 결과 리스트가 최대 8건까지 표시된다.
3. 사용자는 클릭 또는 키보드로 항목을 고른다.
4. 같은 카드 1장이 선택 종목으로 교체된다.
5. 가격이 없으면 `가격 준비중` 상태를 보여준다.

## 7. 기능 요구사항

### 오늘 탭
- `v_home_summary` 단일 행을 기준으로 렌더해야 한다.
- 로드 실패 시 placeholder 숫자를 유지하면 안 된다.

### 검색 결과
- 검색어가 비면 조회하지 않는다.
- 결과는 최대 8건만 노출한다.
- 클릭, Enter, ArrowUp, ArrowDown, Escape를 지원한다.

### 종목 카드
- 먹이 탐색 탭의 카드 수는 1장이다.
- 기본 심볼은 `AAPL`이다.
- 선택 후 카드 교체는 같은 `v_stock_detail_latest` 계약을 사용한다.
- `price_status = missing`이면 `가격 준비중`을 표시한다.
- `price_status = fallback`이면 캐시 가격임을 보조 문구로 알린다.
- KR 가격은 `KRW`, US 가격은 `USD` 포맷을 유지한다.
- `safe_activity_radius_pct`를 `±X.X%` 형식으로 보여준다.
- `safe_activity_level`은 `안전/주의/고위험` 배지로 보여준다.

### 활동반경
- 활동반경은 프론트에서 계산하지 않는다.
- 서버 데이터 계약에서 계산된 값을 그대로 표시한다.
- 활동반경은 투자 추천이 아니라 리스크 가이드다.

## 8. 비기능 요구사항

### 품질
- freshness 기준은 `20시간`
- 대표 품질 게이트 종목은 `AAPL`, `TSLA`, `000660.KS`
- empty summary overwrite를 허용하지 않는다
- 가격이 없어도 카드 빈칸을 허용하지 않는다
- 활동반경은 `1.5% ~ 6.0%` 범위를 유지해야 한다

### 보안
- 브라우저는 `SUPABASE_URL`, `SUPABASE_ANON_KEY`만 사용한다
- `SUPABASE_SERVICE_ROLE_KEY`, `DART_API_KEY`는 브라우저에 노출되면 안 된다
- GPS 같은 개인정보 및 민감 정보는 데이터베이스에 저장하지 않는다

### 운영
- 배치 적재는 `backend/scripts/daily_refresh.py` 기준으로 유지한다
- GitHub Actions 배치 스케줄은 `0 22 * * 0-4` 기준이다
- 브라우저 단계 라이브 폴백은 도입하지 않는다

## 9. 수용 기준
- 오늘 탭이 `v_home_summary` 기준으로 렌더된다
- 오늘 탭이 로드 실패 시 가짜 리스크 숫자를 유지하지 않는다
- 기본 카드가 `AAPL` 기준으로 정상 렌더된다
- 검색 후 카드가 direct-read 계약으로 갱신된다
- 가격이 없는 종목도 카드에서 빈칸 없이 상태가 보인다
- 활동반경 숫자와 단계 배지가 보인다
- 브라우저 네트워크에서 legacy 읽기 API 의존이 없어야 한다

## 10. 현재 릴리스 상태
- direct-read는 구현 완료 상태다
- 가격 상태 UI와 활동반경 실데이터화가 후속 개선으로 반영됐다
- 남은 핵심 리스크는 기능 구현보다 적재 품질 안정성이다
