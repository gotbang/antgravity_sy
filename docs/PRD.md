# AntGravity Supabase Direct-Read PRD

## 작성시각
2026-03-22 13:55:50 KST

## 문서 목적
이 문서는 AntGravity 주식 모드의 현재 제품 요구사항을 direct-read 기준으로 고정한다. 예전처럼 “직결 전환 후보”를 설명하는 문서가 아니라, 이미 구현된 사용자 경험과 그에 필요한 운영 조건을 제품 요구사항 관점에서 정리한다.

## 1. 제품 배경
- 현재 주식 모드는 정적 프론트에서 Supabase 공개 뷰를 직접 읽는 구조다.
- 기존 FastAPI 읽기 경로는 유지되지만 신규 주 경로가 아니라 rollback-only 문맥이다.
- 사용자에게 중요한 건 읽기 경로가 어디냐보다, 오늘 탭과 검색 흐름이 빠르고 안정적으로 보이는지다.

## 2. 해결하려는 문제
- 읽기 경로가 섞이면 화면 기준과 데이터 기준이 흔들린다.
- 검색 결과와 상세 카드가 서로 다른 계약을 쓰면 UX가 불안정해진다.
- 배치 결과와 브라우저 화면 사이에 과도한 폴백이 있으면 실제 데이터 문제를 늦게 발견하게 된다.

## 3. 핵심 사용자 가치
- 오늘 탭에서 시장 요약을 바로 읽을 수 있다.
- 검색 결과를 리스트로 보고 원하는 종목을 직접 고를 수 있다.
- 검색 전 기본 카드와 검색 후 선택 카드가 같은 계약으로 갱신된다.
- 브라우저는 단순한 direct-read 구조를 유지하고, 데이터 품질 문제는 운영 이슈로 분리한다.

## 4. 대상 사용자
- 주식 모드에 진입해 오늘 시장 분위기를 빠르게 확인하려는 사용자
- 종목명을 검색해 대표 종목 한 장의 카드로 빠르게 확인하려는 사용자
- 상세한 트레이딩 화면보다 요약 중심 탐색을 원하는 사용자

## 5. 현재 범위

### 포함
- `v_home_summary` 기반 오늘 탭
- `v_stock_search` 기반 검색 결과 리스트
- `v_stock_detail_latest` 기반 단일 종목 카드
- 기본 심볼 `AAPL`
- 검색 결과 선택 후 동일 카드 갱신
- direct-read 품질 게이트와 rollback-only 운영 문맥

### 제외
- 트렌딩 UI
- 생존 일지 저장/복원
- 로그인/계정
- 실시간 시세 스트리밍
- 브라우저 단계 라이브 폴백

## 6. 핵심 사용자 흐름

### 흐름 1. 오늘 탭 확인
1. 사용자가 주식 모드에 들어간다.
2. 화면은 `v_home_summary`에서 시장 기분, 공포탐욕 점수, 상승/하락 비중, AI 요약을 읽는다.
3. 조회 실패 시 화면은 기본 문구를 유지하고 깨지지 않는다.

### 흐름 2. 기본 카드 확인
1. 사용자가 먹이 탐색 탭을 연다.
2. 기본 카드 한 장이 `AAPL` 기준으로 표시된다.
3. 카드 가격과 요약은 `v_stock_detail_latest` 계약에 따라 렌더된다.

### 흐름 3. 검색 후 종목 교체
1. 사용자가 종목명이나 심볼을 입력한다.
2. 결과 리스트가 최대 8건까지 표시된다.
3. 사용자는 클릭 또는 키보드로 항목을 고른다.
4. 같은 카드 1장이 선택 종목으로 교체된다.

## 7. 기능 요구사항

### 오늘 탭
- `v_home_summary` 단일 행을 기준으로 렌더해야 한다.
- 브라우저는 별도의 서버 폴백 없이 기본 상태를 유지할 수 있어야 한다.

### 검색 결과
- 검색어가 비면 조회하지 않는다.
- 결과는 최대 8건만 노출한다.
- 클릭, Enter, ArrowUp, ArrowDown, Escape를 지원한다.
- 검색 품질은 대표 검색어 기준으로 의미 있는 결과를 상단에 보여줘야 한다.

### 종목 카드
- 먹이 탐색 탭의 카드 수는 1장이다.
- 기본 심볼은 `AAPL`이다.
- 선택 후 카드 교체는 같은 `v_stock_detail_latest` 계약을 사용한다.
- KR 가격은 `KRW`, US 가격은 `USD` 포맷을 유지한다.
- `price`, `summary`가 `null`이어도 화면이 깨지면 안 된다.

## 8. 비기능 요구사항

### 품질
- freshness 기준은 `20시간`
- 대표 품질 게이트 종목은 `AAPL`, `TSLA`, `000660.KS`
- empty summary overwrite를 허용하지 않는다

### 보안
- 브라우저는 `SUPABASE_URL`, `SUPABASE_ANON_KEY`만 사용한다
- `SUPABASE_SERVICE_ROLE_KEY`, `DART_API_KEY`는 브라우저에 노출되면 안 된다
- GPS 같은 개인정보 및 민감 정보는 데이터베이스에 저장하지 않는다

### 운영
- 배치 적재는 `backend/scripts/daily_refresh.py` 기준으로 유지한다
- GitHub Actions 배치 스케줄은 평일 `0 22 * * 1-5` 기준이다
- 원격 최신 반영 여부와 최근 run 성공 여부는 별도 운영 확인 대상이다

## 9. 수용 기준
- 오늘 탭이 `v_home_summary` 기준으로 렌더된다
- 기본 카드가 `AAPL` 기준으로 정상 렌더된다
- 검색 결과 리스트가 direct-read 기준으로 열린다
- `하이닉스` 검색 시 `SK하이닉스 / 000660.KS / KR` 수준의 결과를 고를 수 있다
- 선택 후 카드 가격이 정상 표시된다
- 브라우저 네트워크에서 legacy 읽기 API 의존이 없어야 한다
- 공개 뷰 3개와 품질 게이트가 문서화되어 있어야 한다

## 10. 현재 릴리스 상태
- direct-read는 이미 구현 완료 상태다
- 단일 카드 UI도 이미 반영됐다
- 주요 남은 리스크는 기능 구현이 아니라 적재 품질과 GitHub Actions 운영 확인이다

## 11. 남은 제품 리스크
- KR 적재는 fallback이 들어갔지만 근본 안정성은 더 지켜봐야 한다
- US 적재는 `yfinance` 파일 접근 이슈가 다시 날 수 있다
- 원격 Actions가 최신 코드를 안정적으로 적재하고 있는지는 아직 확인되지 않았다
