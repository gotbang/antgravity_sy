# AntGravity 웹앱 리디자인 계획

## 작성시각
2026-03-21

## 목표
원본 HTML 소스코드의 형식과 내용 구조는 그대로 유지하면서, design 폴더의 디자인 규칙에 맞게 시각적 디자인만 전면 업그레이드한다.

### 1.1 원본 디자인 특징
- 배경색: #F4F1EB (연한 베이지)
- 텍스트: #4A3525 (갈색), #6B5D52 (연한 갈색)
- 강조색: #D4C5B0 (무난한 베이지)
- 카드 라운드: 28px (매우 둥글게)
- 그림자: 0px 4px 16px rgba(0,0,0,0.08)

### 1.2 새 디자인 시스템
- 배경색: #FFF8E7 (크림색)
- 텍스트: #333333 (주요), #666666 (보조), #999999 (부가)
- 강조색: #FF9900 (주황)
- 기능색: #4CAF50 (성공/녹색), #FF5252 (부정/빨강)
- 카드 라운드: 12-16px
- 그림자: 0px 4px rgba(0,0,0,0.1)

### 3.3 컴포넌트별 변경

#### 모드 카드 (로비 화면)
- 배경: 흰색 유지
- 라운드: 28px → 12px
- 그림자: 업데이트
- 활성 상태 테두리: #D4C5B0 → #FF9900

#### 캐릭터 카드
- 배경: 흰색 유지
- 라운드: 28px → 12px
- 말풍선 라운드: 20px → 10px
- 캐릭터 배지: 주황색(#FF9900) 적용

#### 진행 바
- 상승(먹이 발견): 빨강(#FF5252)
- 보합(대열 유지): 주황(#FF9900) + 그라데이션
- 하락(안전 구역): 녹색(#4CAF50)

#### 버튼
- Primary: #FF9900 배경, 흰색 텍스트
- Secondary: 흰색 배경, #FF9900 테두리
- 라운드: 16px → 8-10px
- 그림자 추가

#### 하단 네비게이션
- 활성 탭 배경: #FFF8E7
- 활성 아이콘: grayscale 제거
- 비활성 아이콘: grayscale + opacity 0.5

#### 기분 선택 (감정 일지)
- 선택됨: 녹색 테두리(#4CAF50) + 녹색 배경
- 호버: -translate-y-1

## 7. 실데이터 연동 계획

### 7.1 목표
- 현재 화면에 보이는 시장 데이터 전반을 실제 데이터로 연결한다.
- 구조나 UI 흐름은 유지하고, 데이터 공급과 적재 체계만 실서비스 기준으로 정리한다.
- 1차 범위는 `시장 데이터만 실데이터`로 고정한다.

### 7.2 데이터 소스

#### 한국 데이터
- 공시/재무: `Dartlab/OpenDART`
- 전체 종목 가격/거래대금/시가총액 스냅샷: `PyKRX`

#### 미국 데이터
- 전체 종목 유니버스: `SEC EDGAR company_tickers.json`
- 가격/기초 지표: `yfinance`

### 7.3 저장/적재 원칙
- 원본 진실 소스는 `Supabase`다.
- 보조 파일 캐시는 `비추적 산출물`로만 운영한다.
- 파일 캐시는 git에 커밋하지 않는다.
- GitHub Actions는 `미국장 마감 후 하루 1회` 실행한다.
- 배치 적재 결과는 Supabase upsert를 기준으로 관리한다.

### 7.4 캐시 정책
- 읽기 우선순위는 `in-memory -> Supabase -> 부분 폴백`이다.
- 가격 데이터만 온디맨드 보충을 허용한다.
- 재무/공시/AI 집계 데이터는 배치 적재본만 사용한다.
- 파일 캐시는 배치 결과의 보조 산출물이며 서비스의 기준 원본이 아니다.

### 7.5 실데이터 범위

#### 포함
- 시장 심리
- 트렌딩 종목
- 종목 검색
- 종목 상세
- 감정 지표
- AI 분석 카드

#### 제외
- 개미 광장 실데이터
- 생존 일지 서버 저장
- 로그인
- 민감정보 저장

### 7.6 저장 원칙
- `생존 일지`는 비로그인 기준 `localStorage` 저장으로 유지한다.
- 관심 종목 외 개인 데이터는 서버 저장 대상에 포함하지 않는다.
- GPS 및 기타 민감 개인정보는 저장하지 않는다.

### 7.7 API/서빙 기준
- `GET /api/market-summary`
- `GET /api/market/trending`
- `GET /api/stocks/search`
- `GET /api/stocks/:symbol`

위 API는 모두 Supabase 적재 데이터와 서버 메모리 캐시를 우선 사용한다. 가격 계열 데이터만 부분 폴백을 허용하고, 재무/공시/AI 집계는 배치 기준으로 유지한다.

### 7.8 이전 문서와 충돌하는 오래된 전제 정리
- 이 문서의 실데이터 연동 계획은 기존의 `주황 모바일 리디자인 중심 계획`보다 뒤에 추가된 최신 계획을 우선 기준으로 본다.
- 실데이터 단계에서는 얇은 임시 mock 전제가 아니라 `Supabase 원본 + 배치 적재 + 부분 폴백` 구조를 기준으로 한다.
- diary/journal 전체를 1차 실데이터 대상으로 보지 않는다.

### 7.9 용어 기준
- `Supabase 원본`은 서비스가 우선 조회하는 진실 소스를 뜻한다.
- `파일 캐시 비추적 산출물`은 배치/런타임 보조 결과물이며 git 커밋 대상이 아니다.
- `시장 데이터만 실데이터`는 시장 심리, 트렌딩 종목, 종목 검색/상세, 감정 지표, AI 분석 카드까지만 포함한다.
