# task 실행 초기 백엔드 골격 구축

## 작성시각
2026-03-21 19:50:52 KST

## 해결하고자 한 문제
현재 저장소에는 실데이터 연동용 백엔드, Supabase 적재 구조, API 서빙 레이어가 전혀 없었다. `docs/task.md` 기준으로 최소 실행 가능한 골격을 먼저 만들어 이후 단계가 이어질 수 있게 해야 했다.

## 이번 단계에서 한 일
- `backend/`, `supabase/`, `.github/workflows/` 등 기본 구조 생성
- Python requirements, .env.example, Supabase migration, GitHub Actions 워크플로 추가
- 캐시 정책, Supabase 캐시 접근, KR/US 수집 서비스, market summary 빌더, daily refresh 스크립트 추가
- market/stocks API 라우터와 FastAPI 엔트리 추가
- 최소 smoke 테스트와 partial fallback 테스트 추가
- 프론트 `apiClient.ts`, `live-data.ts`, `live-market-binding.test.ts`, 로딩/빈 상태 CSS 추가
- `docs/task.md`에서 완료된 태스크를 `[X]`로 반영

## 해결된 것
- Setup, Foundational, US1, US2, US3 일부 태스크가 실제 파일 기준으로 반영됐다
- Python 파일 문법 검증이 통과했다
- `pytest backend/tests/test_cache_policy.py backend/tests/test_partial_fallback.py` 2건이 통과했다

## 해결되지 않은 것

- 전체 API 엔드투엔드 검증
