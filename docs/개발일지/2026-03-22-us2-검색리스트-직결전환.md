# us2 검색리스트 직결전환

## 작성시각
2026-03-22 12:29:04 KST

## 해결하고자 한 문제
`US1`까지는 오늘 탭과 기본 카드만 direct-read로 바뀌었고, 검색은 여전히 FastAPI `/api/stocks/search`를 타는 구조였다. `docs/task.md` 기준으로 검색 결과 리스트, 키보드 선택, 카드 1 갱신까지 Supabase 직결 흐름으로 전환할 필요가 있었다.

## 이번 단계에서 한 일
- `src/lib/direct-read-runtime.js`에 검색 controller를 추가해 디바운스, 로딩/빈 상태/에러 상태, 클릭/Enter/방향키/Escape 흐름을 구현했다.
- `index.html`에 검색 결과 패널 DOM을 추가했다.
- `index.html`의 검색 버튼/입력 흐름을 direct-read controller 기반으로 바꿨다.
- `src/tests/us2-search-list.test.ts`, `src/tests/us2-search-keyboard.test.ts`를 추가해 `US2` 회귀 테스트를 만들었다.
- `pnpm lint`, `pnpm typecheck`, `pnpm test`를 통과시켰다.
- `docs/task.md`에서 T018~T023을 완료 처리했다.

## 해결된 것
- 검색 결과가 리스트로 노출된다.
- 클릭과 키보드 선택 흐름이 direct-read runtime 기준으로 정리됐다.
- 검색 선택 후 카드 1이 direct-read detail query로 갱신된다.

## 해결되지 않은 것
- 실제 Supabase env 값은 여전히 비어 있다.
- `US3` 적재 품질 게이트와 `US4` 컷오버/롤백 정리는 아직 남아 있다.
- 트렌딩은 여전히 보류 상태다.

## 메모
검색 버튼 클릭은 direct-read 검색으로 바뀌었고, 기존 `performSearch()`는 controller를 호출하는 얇은 호환 함수 역할만 남겼다.
