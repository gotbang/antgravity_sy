# 적재범위 개편 v2 구현

## 작성시각
2026-03-22 20:28:00 KST

## 해결하고자 한 문제
주식 검색 유니버스는 넓은데 실제 적재는 일부 종목만 돌아가서 `가격 준비중`, `변동 미확인`, `시세 미적재` 상태가 많이 보였다. 이번 단계에서는 direct-read 구조는 유지하면서 적재 범위를 `hot/warm/cold`로 나누고, stale 상태와 실패 원인 기록까지 붙여서 운영 가능한 구조로 바꾸는 게 목표였다.

## 이번 단계에서 한 일
- `backend/core/config.py`에 KRX 인증키와 KRX endpoint 설정을 추가했다.
- `backend/services/kr_market_ingestion.py`를 `KRX key 있으면 공식 KRX`, 없으면 `yfinance_fallback`으로 자동 전환되게 정리했다.
- `backend/scripts/daily_refresh.py`를 phase dispatcher 구조로 재작성했다.
  - `universe-sync`
  - `kr-hot`
  - `us-hot`
  - `warm-rotate`
  - `exact-worker`
  - `scheduled`
- `supabase/migrations/20260321_market_data_foundation.sql`에 coverage/state/failure log/request queue 스키마를 추가했다.
- `supabase/migrations/20260322_public_market_views.sql`에 `coverage_tier`, `freshness_status`, `last_succeeded_at`, `last_attempted_at`, `stale_age_hours` 노출을 추가했다.
- `src/lib/supabase-queries.js`, `src/lib/supabase-adapters.js`를 새 direct-read 메타데이터와 정렬 기준에 맞게 보강했다.
- GitHub Actions workflow를 `scheduled` phase 중심으로 바꿨다.
- 신규 백엔드 테스트와 기존 회귀 테스트를 함께 정리했다.

## 해결된 것
- KR/US 모두 market별 상위 200을 hot으로 분리할 수 있는 기반이 생겼다.
- warm/cold와 stale 상태를 direct-read 뷰에서 읽을 수 있게 됐다.
- exact search 운영 큐와 실패 로그용 스키마가 생겼다.
- KR은 키가 없을 때 `yfinance_fallback`, 키가 생기면 `krx_official`로 코드 수정 없이 전환되게 됐다.
- `pnpm lint`, `pnpm typecheck`, `pnpm test`, `python -m pytest backend/tests`를 통과했다.

## 해결되지 않은 것
- 실제 Supabase에 새 migration을 적용하고 schema cache를 갱신하는 운영 단계는 아직 안 했다.
- KRX 공식 API 키가 없어서 실환경 KRX 호출 검증은 아직 못 했다.
- exact worker 3분 외부 런타임 자체는 저장소 밖 운영 영역이라 이번 단계에서 만들지 않았다.

## 메모
- 현재 workflow는 30분마다 `scheduled` phase를 깨우고, 실제 실행 phase 선택은 스크립트 내부 시간 게이트가 맡는다.
- exact worker는 `--phase exact-worker`를 별도 워커/외부 크론이 3분마다 호출하는 전제로 구현했다.
