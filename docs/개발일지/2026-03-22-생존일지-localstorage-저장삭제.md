# 생존일지 localStorage 저장삭제

## 작성시각
2026-03-22 22:42:54 KST

## 해결하고자 한 문제
생존일지 탭에서 글을 써도 실제 저장이 되지 않았고, 과거 기록도 고정 목업이라 사용자가 직접 남긴 기록을 다시 보거나 지울 수 없었다. 현재 제품 구조가 정적 HTML + Supabase direct-read 기준이라 서버 저장보다는 브라우저 안에서 바로 동작하는 최소 저장/삭제가 필요했다.

## 이번 단계에서 한 일
- 생존일지 저장을 `localStorage` 기반으로 구현했다.
- 선택된 기분 카드의 이모지와 라벨, 작성 날짜, 본문을 함께 저장하게 만들었다.
- 저장된 기록을 과거의 기록 목록에 다시 렌더링하게 만들었다.
- 각 기록에 삭제 버튼을 붙여 실제 삭제와 빈 상태 복귀가 가능하게 만들었다.
- `src/tests/us3-diary-storage.test.ts`를 추가해 저장/삭제 동작을 테스트로 고정했다.

## 해결된 것
- 생존일지에서 글을 쓴 뒤 저장 버튼을 누르면 브라우저에 실제로 저장된다.
- 새로고침 후에도 저장된 기록이 다시 보인다.
- 기록별 삭제 버튼으로 원하는 기록을 지울 수 있다.
- 빈 입력 저장 시 안내 토스트가 뜨도록 처리했다.

## 해결되지 않은 것
- 서버나 Supabase에 동기화되는 구조는 아직 아니다.
- 기기나 브라우저가 바뀌면 기록은 공유되지 않는다.
