# us3 us4 품질게이트 롤백문맥 정리

## 작성시각
2026-03-22 12:34:06 KST

## 해결하고자 한 문제
`US2` 이후에도 direct-read 전환이 안전하다고 보려면 freshness 기준, 대표 종목 품질 게이트, empty summary overwrite 방지, rollback-only 문서화가 필요했다. 이 부분이 빠지면 전환은 되더라도 운영 기준이 모호해진다.

## 이번 단계에서 한 일
- `backend/services/supabase_cache_service.py`의 freshness 기준을 `20시간`으로 맞췄다.
- `backend/scripts/daily_refresh.py`에서 snapshot이 없을 때 `home/trending` summary를 다시 덮어쓰지 않게 조정했다.
- `backend/services/kr_market_ingestion.py`에서 OHLCV-only ticker도 스냅샷 후보에 포함되도록 보강했다.
- `backend/services/market_query_service.py`에 legacy search ranking helper를 추가했다.
- `supabase/migrations/20260322_public_market_views.sql`에 정규화 컬럼을 추가했다.
- `backend/tests/test_cache_freshness_gate.py`, `backend/tests/test_direct_read_quality_gate.py`, `backend/tests/test_search_rank_contract.py`, `backend/tests/test_legacy_read_fallback_contract.py`, `src/tests/us4-cutover-network.test.ts`를 추가했다.
- `docs/plan.md`, `docs/quickstart.md`, `docs/운영가이드/github-secrets.md`를 rollback-only / 20시간 기준으로 갱신했다.
- `docs/task.md`에서 T024~T037, T042를 완료 처리했다.

## 해결된 것
- direct-read 품질 게이트와 rollback-only 문맥이 테스트와 문서 양쪽에 반영됐다.
- freshness 기준이 `20시간`으로 통일되기 시작했다.
- empty summary overwrite 방지와 대표 종목 게이트가 코드로 고정됐다.

## 해결되지 않은 것
- Final Phase placeholder 테스트 정리(T038~T040)는 아직 남아 있다.
- `docs/PRD.md`, `spec/supabase-direct-read.md`, `docs/research.md` 동기화(T043~T045)도 남아 있다.
- 실제 Supabase 환경에서 migration/apply 검증은 아직 하지 않았다.
