# us1 오늘탭 기본카드 직결전환

## 작성시각
2026-03-22 12:20:50 KST

## 해결하고자 한 문제
Foundational까지만 끝난 상태에서는 공개 뷰와 helper는 있어도 실제 화면이 여전히 FastAPI `/api/...`를 읽고 있었다. `US1` 목표대로 오늘 탭과 기본 카드 2개를 Supabase 직결 구조로 옮겨야 했다.

## 이번 단계에서 한 일
- `src/lib/direct-read-runtime.js`를 추가해 오늘 탭과 카드 렌더 로직을 direct-read 런타임 함수로 분리했다.
- `src/tests/us1-home-summary.test.ts`, `src/tests/us1-default-cards.test.ts`를 추가해 `US1` 회귀 테스트를 만들었다.
- `index.html`에 공개 env meta와 import map을 추가하고, 오늘 탭/기본 카드 로딩이 direct-read runtime을 타도록 바꿨다.
- `src/config/public-env.js`를 meta 기반 env 읽기까지 지원하도록 확장했다.
- `src/lib/apiClient.ts`에 레거시 read helper 문맥을 주석으로 명확히 남겼다.
- `pnpm lint`, `pnpm typecheck`, `pnpm test`를 다시 돌려 통과시켰다.
- `docs/task.md`에서 T013~T017을 완료 처리했다.

## 해결된 것
- 오늘 탭과 기본 카드 2개가 더 이상 FastAPI 읽기 함수에 직접 의존하지 않게 됐다.
- direct-read UI 로직이 테스트 가능한 모듈로 분리됐다.
- `US1` 범위는 lint/typecheck/test 기준으로 검증 가능한 상태가 됐다.

## 해결되지 않은 것
- 실제 Supabase 공개 env 값은 아직 비어 있다.
- 검색 리스트와 종목 선택 흐름은 아직 FastAPI 검색 경로를 사용한다.
- 레거시 `/api/...` 호출 제거는 `US2`, `US4`까지 가야 완전히 끝난다.

## 메모
현재 `index.html`은 direct-read 모듈을 사용하도록 바뀌었지만, env 값이 없으면 기본 UI를 유지하고 에러만 남기는 안전한 상태로 두었다.
