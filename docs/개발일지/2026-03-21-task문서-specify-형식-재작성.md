# task문서 specify 형식 재작성

## 작성시각
2026-03-21 19:50:52 KST

## 해결하고자 한 문제
기존 `docs/task.md`는 자유형 체크리스트라 바로 실행 가능한 작업 문서로 쓰기 어려웠다. `docs/plan.md` 7장의 실데이터 연동 계획을 기준으로 `T001`, `US1` 형식의 실행형 task 문서가 필요했다.

## 이번 단계에서 한 일
- 기존 `docs/task.md`를 전면 교체
- `US1 적재/캐시`, `US2 API 서빙`, `US3 프론트 연결` 구조로 재작성
- 모든 태스크를 `- [ ] T### ...` 형식으로 통일
- 의존성, 병렬 실행 예시, MVP 전략까지 문서에 추가

## 해결된 것
- 구현자가 바로 착수 가능한 task 문서가 생겼다
- 실데이터 연동 범위를 리디자인과 분리해서 읽을 수 있게 됐다
- `시장 데이터만 실데이터`, `Supabase 원본`, `파일 캐시 비추적 산출물` 기준이 task 문서에도 고정됐다

## 해결되지 않은 것
- 실제 백엔드/프론트 구현
- `.specify` 템플릿 부재로 인한 자동 생성 워크플로

## 메모
이번 단계는 문서만 수정했고 소스코드는 변경하지 않았다.
