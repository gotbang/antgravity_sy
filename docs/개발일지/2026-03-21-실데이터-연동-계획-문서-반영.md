# 실데이터 연동 계획 문서 반영

## 작성시각
2026-03-21 19:45:34 KST

## 해결하고자 한 문제
기존 `docs/plan.md`는 디자인 리디자인 중심이라 이후 실데이터 연동 구현 기준으로 쓰기 어려웠다. 구현 전에 문서만으로 데이터 소스, 적재 구조, 캐시 정책, 구현 순서를 확정할 필요가 있었다.

## 이번 단계에서 한 일
- `docs/plan.md` 하단에 `실데이터 연동 계획` 섹션 추가
- `docs/task.md` 생성
- 실데이터 범위를 `시장 데이터만`으로 고정
- 데이터 소스, Supabase 원본 원칙, 파일 캐시 비추적 산출물 원칙, 배치 시점을 문서화

## 해결된 것
- plan 문서에 실데이터 연동 기준이 추가됐다
- 이후 구현자가 따라갈 `task.md`가 생겼다
- 개미 광장과 생존 일지 서버 저장이 1차 범위에서 제외됨이 명확해졌다

## 해결되지 않은 것
- 실제 Supabase 스키마 구현
- 실제 수집 스크립트 작성
- 실제 API/프론트 연결

## 메모
이번 단계는 문서만 수정했고 소스코드는 변경하지 않았다.
