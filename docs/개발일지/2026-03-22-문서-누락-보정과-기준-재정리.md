# 문서 누락 보정과 기준 재정리

## 작성시각
2026-03-22 13:55:50 KST

## 해결하고자 한 문제
`memory.md`, `research.md`, `docs/`를 기준으로 현재 상태를 다시 읽어보니, 구현은 이미 끝난 direct-read 구조인데 `docs/plan.md`는 아직 전환 검토 문맥과 과거 부채 문맥이 많이 남아 있었다. `docs/PRD.md`와 `spec/supabase-direct-read.md`도 현재 기준 문서로 쓰기엔 너무 얇아서, 다음 세션에서 읽을 기준이 흔들릴 수 있었다.

## 이번 단계에서 한 일
- `memory.md`, `research.md`, `docs/PRD.md`, `docs/plan.md`, `docs/task.md`, `docs/quickstart.md`, `docs/research.md`, `docs/운영가이드/github-secrets.md`, `spec/supabase-direct-read.md`를 다시 읽고 현재 구현 기준을 정리했다.
- `docs/plan.md`를 전환 검토 문서가 아니라 현재 운영 기준 문서로 재작성했다.
- `docs/PRD.md`를 placeholder 수준에서 현재 사용자 흐름과 수용 기준이 드러나는 요구사항 문서로 확장했다.
- `spec/supabase-direct-read.md`를 공개 뷰, 프론트 계약, 보안 경계, rollback 계약이 보이는 실제 스펙 문서로 확장했다.
- 이번 문서 정리 작업 자체를 개발일지와 프롬프트 문서로 남겼다.

## 해결된 것
- `docs/plan.md` 안에 남아 있던 `docs/PRD.md` 부재, `spec/` 부재 같은 과거 문맥을 제거했다.
- plan/PRD/spec이 모두 현재 구현 완료 상태를 기준으로 읽히게 됐다.
- 단일 카드 UI, direct-read 주 경로, rollback-only FastAPI 문맥이 세 문서에서 일관되게 맞춰졌다.

## 해결되지 않은 것
- GitHub Actions 원격 반영 여부와 최근 run 성공 여부는 여전히 확인하지 못했다.
- KR/US 적재 안정성 리스크 자체는 문서화만 했고 해결하지는 않았다.
- 코드 구현은 하지 않았다.

## 메모
이번 단계는 사용자 요청대로 문서만 수정했고, 구현 파일은 건드리지 않았다.
