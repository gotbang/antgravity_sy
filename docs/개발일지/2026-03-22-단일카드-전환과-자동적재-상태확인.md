# 단일카드 전환과 자동적재 상태확인

## 작성시각
2026-03-22 13:38:03 KST

## 해결하고자 한 문제
아슬아슬 나뭇가지 화면에 종목 카드가 두 장 보이는 구조를 한 장으로 단순화해야 했고, 가격이 비는 종목들이 많은 이유가 GitHub Actions 자동 적재 문제인지 적재 코드 문제인지 함께 확인해야 했다.

## 이번 단계에서 한 일
- `index.html`에서 두 번째 종목 카드를 제거하고 초기 `TSLA` 카드 로드를 없앴다.
- `src/tests/us1-default-cards.test.ts`를 단일 카드 기준으로 맞췄다.
- `pnpm lint`, `pnpm typecheck`, `pnpm test`를 다시 통과시켰다.
- `pnpm run sync:public-env` 후 Playwright로 실제 브라우저에서 카드 개수와 가격 표시를 확인했다.
- 개발일지와 프롬프트에서 적재/자동화 관련 기록을 다시 읽고 현재 workflow 정의를 확인했다.

## 해결된 것
- 먹이 탐색 탭의 종목 카드는 한 장만 보이게 됐다.
- 실제 브라우저에서 `cardCount = 1`, `card2Exists = 0`, `card1Price = US$247.99`를 확인했다.

## 해결되지 않은 것
- GitHub Actions의 실제 원격 실행 이력은 이 로컬 저장소만으로는 확인하지 못했다.
- 로컬 저장소엔 git remote가 없어서 현재 이 수정이 원격 Actions에 반영돼 있는지는 별도 확인이 필요하다.

## 메모
현재 코드 기준 자동 적재 워크플로 파일은 존재하지만, 가격 이슈의 직접 원인은 적재 로직 안정성 문제였고 그 중 KR `PyKRX` fallback 문제는 이번 세션에서 이미 수정했다.
