# 프론트 접근성 보안 강화

## 작성시각
2026-03-22 23:37:41 KST

## 해결하고자 한 문제
프론트 코드 리뷰 과정에서 접근성과 보안 측면의 실제 이슈가 확인됐다. viewport에서 브라우저 확대를 막고 있었고, 생존일지 기분 선택 UI는 키보드 접근이 어려웠으며, 검색 결과와 생존일지 렌더에 `innerHTML` 문자열 조합이 남아 있어 XSS 경로가 열려 있었다. backend의 `/api/health`도 실행 환경값을 그대로 노출하고 있었다.

## 이번 단계에서 한 일
- viewport의 `user-scalable=no`를 제거했다.
- 생존일지 기분 선택을 radio 입력 기반으로 바꿔 키보드 접근 가능하게 만들었다.
- textarea에 명시 label을 연결했다.
- 검색 결과와 생존일지 기록 렌더를 `createElement`와 `textContent` 기반으로 바꿨다.
- `/api/health` 응답을 `status`만 남기는 최소 payload로 줄였다.
- 검색 결과 XSS 회귀 테스트, 생존일지 XSS 회귀 테스트, health API 테스트를 추가했다.
- 가격 포맷터 캐시와 검색 활성 상태 갱신 최적화도 같이 반영했다.

## 해결된 것
- 확대 금지 접근성 문제가 사라졌다.
- 기분 선택이 키보드 접근 가능한 구조가 됐다.
- 생존일지 입력이 보조기기 기준으로 더 명확해졌다.
- 검색 결과와 생존일지 기록에서 문자열 삽입 기반 XSS 위험을 줄였다.
- health 응답의 불필요한 환경 노출을 제거했다.

## 해결되지 않은 것
- 저장소 전체 기준으로는 여전히 다른 unrelated 워킹트리 변경이 남아 있다.
- backend 전반 보안 검토는 이번 단계에서 전체 범위로 확장하지 않았다.
