## 프롬프트 내용

### 1차 프롬프트
```
research.md 보고 만들려는 웹앱 구조를 완벽히 파악한건가? 파악했을때만 ok. 조금이라도 모르는거면 질문. 참고로 우리가 만들려는 프로그램의 ui 레퍼런스 이미지들이야. 형식이나 내용 구조는 전혀 바꾸지말고 그대로 구현하는데 디자인만 design폴더안의 규칙에 맞게 전면 업그레이드할거야. 완벽히 이해한 내용을 어떻게 구현할건지 plan.md에 자세하기 기록
```

### 2차 프롬프트 (소스코드 제공)
```
이게 원래 웹앱 소스코드다. 그런데 디자인을 design폴더안의 문서들을 보고 거기에 맞게 완전히 변경할거야
```

## 사용된 스킬
- design-sample-master: 디자인 시스템 생성

## 결과
- docs/plan.md: 구현 계획 문서
- index.html: 새 디자인 시스템이 적용된 웹앱
- 색상, 라운드, 그림자 등 디자인 토큰 전면 교체
- HTML 구조 및 JavaScript 로직 유지

## 교훈
- 원본 소스코드가 있으면 구조 유지하면서 디자인만 변경하는 것이 효율적
- plan.md로 변경 사항을 명확히 매핑하면 작업이 체계적
- 색상 매핑 테이블이 있으면 누락 없이 변경 가능
